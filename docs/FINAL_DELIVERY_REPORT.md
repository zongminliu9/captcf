## 2. What is implemented (verified)

- **Guest-first loop**: home → real question in ≤2 clicks → instant feedback with distractor
  rationales + transcript → per-skill unofficial estimate (0–699/0–20, CEFR, NCLC, confidence) →
  full review. No signup required.
- **Guest → account merge**: idempotent, lossless (unit + integration + E2E verified).
- **Practice engine**: quick / by-skill / custom / mistakes / bookmarks / due-review; session state
  machine; autosave; refresh/reopen-safe; **server-authoritative grading**.
- **Listening**: 108 items with **real French-Canadian multi-voice audio**; player with
  play/pause/replay/speed/transcript (study) and limited-plays / no-transcript (exam).
- **Reading**: 126 original items A1–C2.
- **Writing**: 45 tasks (3 official types); editor with autosave/word-count/timer/focus; **deterministic
  local rubric + unofficial 0–20 band + model answer** (no AI key).
- **Speaking**: 45 tasks; **real recording** with volume metering, two-take comparison, self-eval,
  **permission fallback**; local rubric (no ASR).
- **Mock exams**: **4 complete forms** (39 CO + 39 CE + 3 EE + 3 EO each); timed CO+CE runner with
  **refresh-safe per-section clocks** and **auto-submit**; per-section results.
- **Adaptivity**: transparent mastery per skill/subtype; SM-2 spaced repetition; rules-based,
  explainable recommendation engine with machine-readable reason codes.
- **Entitlements**: Free vs Premium gating; **dev simulator** (Premium without Stripe) + Stripe adapter.
- **Admin**: question list/filter/edit/status/versioning, **atomic JSON import** with validation,
  issue reports.
- **i18n**: fr / en / zh with safe French fallback. **Privacy**: data export + account deletion.
- **A11y & performance**: WCAG-oriented markup; Lighthouse 100/96/96/100 on public pages.

## 3. Architecture (summary)

Next.js 16 (App Router, RSC + Server Actions + Route Handlers) · React 19 · TypeScript strict ·
Tailwind v4 · Drizzle ORM · PostgreSQL 17 · Zod 4 · custom cookie-session auth (bcryptjs) ·
Biome · Vitest · Playwright. Pure domain logic (exam/mastery/review/recommend/entitlements/
writing/speaking/assemble) is DB-free and unit-tested. Full details: `docs/ARCHITECTURE.md`.

### Database (34 tables)

Identity (`users`, `auth_sessions`, `guest_sessions`, `accounts`); learner (`profiles`,
`exam_goals`, `study_plans`); content (`questions`, `options`, `audio_assets`,
`question_versions`, `writing_tasks`, `speaking_tasks`, `vocabulary_items`, `mock_tests`,
`mock_sections`, `mock_items`); activity (`practice_sessions`, `responses`, `attempts`); learning
state (`mastery_records`, `review_queue`, `mistakes`, `bookmarks`, `vocabulary_progress`);
productive (`writing_submissions`+feedback, `speaking_submissions`+feedback); commerce/ops
(`subscriptions`, `entitlements`, `usage_counters`, `issue_reports`, `analytics_events`,
`content_audits`). Owner CHECK (user XOR guest), per-owner unique indexes, FKs, indexes.

## 7. Current real limitations

1. **Content below the aspirational scale** (108/126 vs 200/250); pipeline ready to scale.
2. **EE/EO are not inside the timed mock runner** — they use their dedicated tools (same rubrics)
   and contribute to the record; the timed runner covers the auto-scored CO+CE. (Top follow-up.)
3. **Recommendation/mastery are rules-based** (by design, for transparency), not ML.
4. **Writing/Speaking AI grading and speech-to-text** are adapter stubs (local rubrics only) until
   provider keys are set. No LanguageTool integration yet.
5. **Password reset** is a local-dev concept only (no email delivery configured).
6. Lighthouse measured on **public** pages (authed pages need a session).

## 8. Top 3 next iterations

1. **Fully embed EE + EO in the timed mock runner** (inline editor/recorder per section) and add
   productive scores to the mock result, for a complete single-flow 4-section exam.
2. **Scale + broaden content** to the aspirational minimums with the same author→reviewer pipeline,
   plus multi-question reading passages and a richer C1/C2 pool.
3. **Wire the AI/ASR adapters** (writing & speaking feedback, transcription) and LanguageTool for
   deeper, still-clearly-unofficial feedback.
